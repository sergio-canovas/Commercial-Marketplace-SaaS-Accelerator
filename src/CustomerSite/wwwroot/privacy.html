1. Who We Are

This Privacy Policy applies to the dForts Copilot Readiness Monitor application (the “Product”), published by dForts (“we”, “us”, “our”) on Microsoft AppSource and distributed as a SharePoint Framework (SPFx) solution.

Contact: privacy@dforts.com  |  dforts.com

2. Scope

This policy describes how we collect, use, and protect information when your organization installs and uses the dForts Copilot Readiness Monitor. It applies to all end users within the subscribing Microsoft 365 tenant.

This Product operates primarily within your own Microsoft 365 tenant. The only outbound data transfer to dForts infrastructure is the license validation request described in Section 4.

3. Data Processed Within Your Tenant

The following data is accessed at runtime to perform the oversharing analysis. It is never transmitted to dForts servers and is processed exclusively within your Microsoft 365 environment:

Data type Purpose Storage
SharePoint site URLs, names, and IDs Enumerate sites for tenant-wide report (admin web part) In-memory only, not persisted
File and folder names, URLs, and item IDs Identify items with risky permission configurations In-memory only, not persisted
SharePoint permission assignments (role assignments, sharing links) Detect oversharing: Everyone, anonymous, external, and unique permissions In-memory only, not persisted
Microsoft 365 group membership counts Estimate exposure impact when a group has unique permissions In-memory only, not persisted
Site owner display names and email addresses Populate the “Notify Owners” feature in the admin report In-memory only, not persisted
Scan results summary (score, risky item list, remediation log) Cache results for performance; display history and audit trail to site owners Stored in a hidden SharePoint list within your own site
Acknowledged items Track permissions intentionally accepted by the site owner Stored in a hidden SharePoint list within your own site

All SharePoint list data is stored inside your own tenant and is subject to your organization’s own data governance and retention policies.

4. Data Transmitted to dForts

The only information sent to dForts infrastructure is:

  • Your Microsoft 365 Tenant ID (GUID) — transmitted via HTTPS to dfortsspcp01.portal.azurewebsites.net at web part load time to verify whether your organization holds an active license. This identifier is a non-personal, organization-level identifier assigned by Microsoft.

This request returns only the license plan tier (standard, premium, or none) and the subscription expiry date. No personal data, file contents, or SharePoint metadata are transmitted.

The result is cached in the user’s browser session storage for up to one hour to avoid repeated API calls.

5. Microsoft Graph Permissions

The Product requests the following Microsoft Graph delegated permissions, which must be approved by a Global Administrator or SharePoint Administrator at installation time:

Permission Justification
Sites.Read.All Enumerate SharePoint sites for the tenant-wide report
Files.Read.All Read file and folder metadata to detect permission anomalies
GroupMember.Read.All Count transitive members of Microsoft 365 groups with unique permissions
Group.Read.All Read group owners to populate the “Notify Owners” feature
User.Read.All Resolve user identities and email addresses for owner notifications
Mail.Send Send notification emails to site owners on behalf of the IT administrator

All permissions are exercised under the delegated identity of the signed-in user. The Product does not use application-level (app-only) permissions.

6. Data Retention

  • Browser session storage (license cache): cleared when the browser tab is closed.
  • SharePoint hidden lists (scan cache, acknowledgements, remediation log): retained indefinitely until the site owner or administrator deletes them. These lists reside in your own tenant.
  • dForts license database: stores Tenant ID, active plan, and subscription dates for the duration of the subscription plus any applicable legal retention period. Deleted upon written request after subscription cancellation.

7. Legal Basis for Processing (GDPR)

For organizations in the European Economic Area, the legal basis for processing is:

  • Contract performance (Article 6(1)(b) GDPR) — processing necessary to deliver the licensed service.
  • Legitimate interests (Article 6(1)(f) GDPR) — license validation to protect against unauthorized use.

8. Data Subject Rights

Depending on your jurisdiction, individuals may have the right to access, correct, or erase personal data we hold. Since almost all data is processed within your own tenant, the primary point of control is your organization’s Microsoft 365 administrators.

For data held in dForts infrastructure (Tenant ID, subscription details), contact us at privacy@dforts.com. We will respond within 30 days.

9. Security

All communication between the Product and dForts infrastructure uses HTTPS (TLS 1.2 or higher). The license API endpoint does not require authentication beyond the Tenant ID and does not expose sensitive data in its response. SharePoint data never leaves your tenant.
